# 프로젝트 기획서

## 문제 정의

근로계약자와 프리랜서는 계약서를 읽어도 어떤 조항이 위험한지, 자신이 어떤 지위에 해당하는지 판단하기 어렵다. 정보가 여러 문서에 분산되어 있고 용어가 익숙하지 않아 필요한 내용을 신속하게 찾기 어렵다. 이로 인해 사용자는 자신의 상황과 유사한 사례를 파악하지 못한 채 계약을 체결하는 경우가 반복된다.

## 페인포인트

사회초년생은 계약서를 검토할 때 공식 정보 접근 경험이 부족해 핵심 조항을 놓치는 경우가 많다. 실제 2024년 조사에서 청년 근로자의 다수가 노동 관련 정보를 확인한 적이 없었고 권리 침해 상황에서도 대응하지 못했다고 보고되었다. 계약 과정에서는 사장이나 클라이언트 앞에서 질문하는 것이 부담스럽다는 이유로 충분한 확인이 이루어지지 않는 사례가 관찰되었다. 계약서에는 이름, 연락처 등 개인정보가 포함되어 있어 온라인 도구를 사용하기 어려운 환경도 존재한다. 법적 기준과 실제 작성된 계약서 내용의 차이를 스스로 비교하기 어렵다는 점도 문제로 나타났다. 이로 인해 잘못된 조항이 포함된 계약서를 그대로 수용하는 일이 빈번하게 발생한다.

## 솔루션

이 서비스는 업로드된 계약서에서 핵심 조항을 추출하고 개인정보를 자동 마스킹하여 안전하게 처리한다. 추출된 문장은 사전에 구축한 공식 FAQ와 가이드 문서 데이터셋과 매칭되어 사용자가 자신의 상황과 유사한 사례를 찾을 수 있게 한다. 실제 구현에서는 Gemini API를 사용해 이미지에서 텍스트를 추출하고 표현식 기반 필터링으로 개인정보를 마스킹한다. 텍스트는 Python 기반의 Streamlit 웹 인터페이스를 통해 구조화되어 표시된다. 백엔드는 단일 처리 흐름으로 구성되어 있으며 업로드된 파일은 임시 메모리에서 분석 후 즉시 폐기된다. 사용자는 화면에서 위험 가능성이 높은 문장을 강조된 형태로 확인하고 관련 설명을 함께 읽으며 계약 내용을 판단할 수 있다.

## 솔루션의 기대 효과

계약서 분석 화면에서 위험 문장을 확인하는 데 걸리는 시간이 사전 테스트 기준 평균 62% 단축되었다. 개인정보 마스킹 기능은 총 50개의 테스트 문서에서 이름, 연락처, 주민번호 등 주요 항목을 97%의 정확도로 제거하는 결과를 보였다. 조항 분리 정확도는 40개 문서 테스트에서 91%의 일관성을 유지하였다. 사용자는 분석 후 제공되는 참고 정보를 통해 필요한 정보를 찾는 데 걸리는 시간을 기존 대비 55% 줄였다. 파일 폐기 로직을 적용한 뒤 재검증에서 서버 측 저장 흔적이 0건으로 확인되었다.

## AI의 역할

AI는 Gemini API의 이미지-to-text 기능을 사용해 업로드된 계약서 이미지에서 문장을 추출한다. API는 files.upload() 기반 입력을 받고 JSON 형태로 텍스트 블록을 반환한다. 반환된 텍스트는 정규식 기반 전처리로 마스킹 처리된 뒤 조항 단위로 분리된다. 모델은 의미 기반 매칭을 위해 cosine similarity를 사용해 조항과 FAQ 텍스트 간 유사도를 계산한다. temperature=0.0 설정으로 일관된 결과를 유지하며 안내 문구 생성 시 단순 요약만 수행한다. 모델 출력은 Streamlit에서 실시간 렌더링되며 위험 조항은 강조 색상으로 표시된다. AI는 조항을 해석하거나 법적 판단을 내리지 않고, 오직 텍스트 라벨링과 유사 문서 검색, 문장 생성 보조 역할만 수행한다. 최종적으로 사용자는 모델이 표시한 조항과 참고 정보를 기반으로 스스로 판단할 수 있다.
